# F4.2 Scale

## Tổng quan

4 thay đổi infrastructure khi hệ thống tăng tải: Redis queue, Cloudflare R2 storage, PostgreSQL connection pooling, monitoring. Tất cả đã được chuẩn bị từ đầu (abstraction layer) — chỉ đổi config + .env, không sửa application code.

---

## Trigger để scale

| Dấu hiệu | Action |
|---------|--------|
| Queue lag > 5 phút hoặc >200 jobs/ngày | Chuyển sang Redis queue + Horizon |
| Disk VPS >80% hoặc storage >10 GB | Chuyển sang Cloudflare R2 |
| DB CPU >70% liên tục | Thêm PgBouncer connection pooling |
| Error rate tăng, không có visibility | Cài Sentry + health check |

### Config

```php
// config/filesystems.php
'disks' => [
    'local' => [
        'driver' => 'local',

        'region'   => 'auto',            // R2 không dùng region chuẩn

        'use_path_style_endpoint' => false,
        'visibility' => 'private',        // mặc định private, signed URL cho backup download
```

### Migration Command — copy local files lên R2

```php
// app/Console/Commands/MigrateStorageToR2.php
class MigrateStorageToR2 extends Command
{
    protected $signature = 'storage:migrate-to-r2 {--dry-run}';

    public function handle(): void
    {
        $local = Storage::disk('local');
        $r2    = Storage::disk('r2');

        $files = $local->allFiles();
        $bar   = $this->output->createProgressBar(count($files));

        foreach ($files as $file) {
            if (!$this->option('dry-run')) {
                $r2->put($file, $local->get($file));
            }
            $bar->advance();
        }

        $bar->finish();
        $this->newLine();
        $this->info(count($files) . ' files migrated.');
    }
}
```

```bash
# Chạy dry run trước
php artisan storage:migrate-to-r2 --dry-run

# Migrate thực
php artisan storage:migrate-to-r2

# Đổi .env FILESYSTEM_DISK=r2
# Test + verify
# Xóa local files sau khi confirmed OK
```

### Signed URL cho private files (backup download)

Hiện tại `BackupController::download()` dùng `Storage::disk()->download(...)` — cần đổi sang signed URL khi dùng R2:

```php
// app/Http/Controllers/Dashboard/BackupController.php
public function download(Event $event, string $token): RedirectResponse
{
    $this->authorize('update', $event);

    $backup = EventBackup::where('event_id', $event->id)
        ->where('token', $token)
        ->where('status', 'ready')
        ->firstOrFail();

    abort_if($backup->expires_at && $backup->expires_at->isPast(), 410, __('backup.link_expired'));

    // Tạo signed URL thay vì stream — giảm tải server
    $signedUrl = Storage::disk()->temporaryUrl(
        $backup->zip_path,
        now()->addMinutes(5),
    );

    return redirect()->away($signedUrl);
}
```

## 3. PostgreSQL Connection Pooling

### PgBouncer (nếu self-host)

PgBouncer pool mode `transaction` — phù hợp nhất với Laravel (không dùng prepared statements qua pool):

```ini
; /etc/pgbouncer/pgbouncer.ini
[databases]
invia = host=localhost port=5432 dbname=invia

[pgbouncer]
listen_port = 6432
listen_addr = 127.0.0.1
auth_type = md5
pool_mode = transaction
max_client_conn = 200
default_pool_size = 20
```

```env
# Trỏ DB_PORT sang PgBouncer
DB_HOST=127.0.0.1
DB_PORT=6432
```

### Read replica (nếu dùng managed DB)

```php

    'port'     => env('DB_PORT', '5432'),
    'database' => env('DB_DATABASE', 'invia'),
    'username' => env('DB_USERNAME', 'postgres'),
    'password' => env('DB_PASSWORD', ''),
    'charset'  => 'utf8',
    'prefix'   => '',
    'schema'   => 'public',
    'sslmode'  => 'prefer',
```


Laravel tự route SELECT → read, INSERT/UPDATE/DELETE → write.

## 4. Monitoring

### Health Check Endpoint

```php
// routes/web.php
Route::get('/health', function () {
    $checks = [];

    // DB
    try {
        DB::select('SELECT 1');
        $checks['db'] = 'ok';
    } catch (\Throwable) {
        $checks['db'] = 'error';
    }

    // Redis (nếu dùng)
    if (config('queue.default') === 'redis') {
        try {
            Redis::ping();
            $checks['redis'] = 'ok';
        } catch (\Throwable) {
            $checks['redis'] = 'error';
        }
    }

    // Storage
    try {
        Storage::disk()->exists('health-check-probe');
        $checks['storage'] = 'ok';
    } catch (\Throwable) {
        $checks['storage'] = 'error';
    }

    $status = in_array('error', $checks) ? 503 : 200;

    return response()->json(['status' => $status === 200 ? 'ok' : 'degraded', 'checks' => $checks], $status);
})->name('health');
```

## Sequence: Local → R2 → CDN

```
File upload (OG image, music, backup)
  │
  └─ Storage::disk()->put(path, contents)
       │
       ├─ FILESYSTEM_DISK=local  → storage/app/...
       │
       └─ FILESYSTEM_DISK=r2    → R2 bucket
            │
            └─ CDN (Cloudflare)
                 └─ assets.invia.vn/{path}
```

Files cần URL public (thumbnails, music preview):
```php
Storage::disk()->url($path)
// local: /storage/{path}
// r2:    https://assets.invia.vn/{path}
```

Files cần URL có TTL (backup download):
```php
Storage::disk()->temporaryUrl($path, now()->addMinutes(5))
// r2: signed URL từ R2 API
// local: không hỗ trợ temporaryUrl → cần custom driver hoặc serve qua controller
```

> **Note:** `temporaryUrl()` trên `local` disk throw `RuntimeException`. Khi còn dùng local (dev/staging), `BackupController::download()` phải fall back về `Storage::disk()->download()`.

## Edge Cases

| Tình huống | Xử lý |
|-----------|-------|
| Redis down khi queue = redis | Jobs fail + retry theo `$tries` — không mất data nếu job được serialize đúng |
| R2 upload fail | `Storage::put()` throw exception → job retry. Backup job có `$tries = 2` |
| R2 signed URL hết hạn 5 phút | Host click download lại → `BackupController::download()` tạo URL mới |
| PgBouncer `transaction` mode và Laravel transactions | Tương thích — PgBouncer giữ connection suốt transaction block |
| PgBouncer và `SET` statement | PgBouncer `transaction` mode reset session vars — không dùng `SET` bên ngoài transaction |
| Read replica lag khi viết xong đọc lại | Dùng `DB::connection('mysql_write')` hoặc `DB::statement()` cho critical reads sau write |
| Telescope trên production | Không enable (`APP_ENV=production`) — chỉ dev |
| `/health` bị flood | Rate limit hoặc đặt sau load balancer health check (không expose public) |
| Migrate storage trong khi có traffic | Down time không cần — copy lên R2 xong rồi mới đổi FILESYSTEM_DISK |
